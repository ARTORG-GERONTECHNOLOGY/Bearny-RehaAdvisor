# Security Policy

## Supported Versions

Security fixes are prioritized for the latest release on `main` and production deployments using the current release tag.

## Reporting A Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Use one of these private channels:

1. GitHub Security Advisories (preferred):
   - Open a private report in the repository "Security" tab.
2. Email:
   - `noora.angelva@unibe.ch`
   - Subject line: `Security report: RehaAdvisor`

Please include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment (confidentiality, integrity, availability)
- Any suggested mitigation if available

## Response Process

- Initial acknowledgment target: within 3 business days
- Triage and severity assessment: as soon as reproducible details are available
- Remediation timeline: based on severity and operational risk
- Disclosure: coordinated disclosure after a fix is available

## Safe Harbor

We support good-faith security research and will not pursue action against reports made responsibly, privately, and without data destruction or service disruption.
